Lambda Lifting

The Task

In this project you are asked to write a one person game using Java Swing along the lines of the game Boulder Dash. Specifically we use the rules taken from a recent ICFP contest.

It is important that the game/simulation be run visually and manually, run visually and automatically, and automatically without visualization. 

java -Ddisplay=graphics Lift ./contest1.map               # manually
java -Ddisplay=graphics Lift ./contest1.map < input.txt   # predetermined moves
java -Ddisplay=graphics Lift ./contest1.map RandomWalk    # play automatically
java -Ddisplay=none     Lift ./contest1.map RandomWalk    # automatically, score only
java -Ddisplay=text     Lift ./contest1.map RandomWalk    # automatically, text output

The first argument to Java is manditory and is the name of a file or a URL containing a US-ASCII map of the mine. The second argument is optional and if present signals a Java program which will act as the agent to direct the miner in solving the map. Otherwise the moves come from the standard input stream.

The default value for -Ddisplay is none. In all cases, the program should print the integer score to the standard output stream. In the case of graphics and none, this is the only output on the standard output stream. In the case of text, board and the score is printed to the standard output with each move.

Moves

The moves are A (abort), U (up), D (down), L (left), R (right), and W (wait); illegal moves are equivalent to 'W'.

Assumptions

In order to run a "miner" -- a mine searching program -- the main program must make some assumptions about the miner. A mine searching program is required to have a static method called next. The method next returns a character representing the move of the miner, one of AUDLRW. The method takes three arguments: the row that the miner is currently in, the column that the miner is currently in, and the entire mine represented as a two dimensional character array. Use reflection to find and all the next method of the miner. 
 String methodName = "next";
 Class[] parameters = new Class[] {int.class, int.class, char[][].class};

Here is a java class for a miner that gets its direction by asking the user: DeaultMiner.java. Note the main program must protect itself from possible abuse by a miner program by passing a copy of the map.
